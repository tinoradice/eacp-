Funcionamiento del sistema

Detección, análisis, alerta y respuesta

Cómo trabaja SentinelGas paso a paso

SentinelGas convierte datos del entorno en información útil para prevenir accidentes, activar una advertencia rápida y asistir al usuario con una respuesta clara.

Arquitectura del sistema

El proyecto está compuesto por sensores de detección, un módulo de procesamiento, un sistema de alertas y una aplicación móvil conectada al usuario.

1️⃣

Monitoreo del ambiente

Los sensores instalados analizan continuamente el aire para detectar concentraciones peligrosas de gas o monóxido de carbono.

2️⃣

Lectura de datos

Los valores registrados son enviados al microcontrolador, que interpreta la información recibida y la compara con rangos predefinidos.

3️⃣

Análisis del riesgo

Si el nivel medido supera el umbral seguro, el sistema reconoce que existe una situación anormal y activa el protocolo de advertencia.

4️⃣

Alerta inmediata

Se activa una alarma local y al mismo tiempo se envía una notificación a la app para informar qué tipo de riesgo fue detectado.

Respuesta del sistema

SentinelGas no solo detecta, sino que acompaña al usuario con información útil para tomar la mejor decisión según el tipo de peligro.

🪟

Si hay fuga de gas

El sistema recomienda ventilar el ambiente, evitar fuentes de ignición y mantener distancia de posibles puntos de riesgo.

☠️

Si hay monóxido

La aplicación alerta sobre un gas peligroso difícil de percibir y sugiere evacuar el lugar para reducir el riesgo de intoxicación.

📲

Comunicación con la app

El usuario puede visualizar el estado del sistema, recibir advertencias y comprender rápidamente qué está ocurriendo en el entorno.

🛡️

Objetivo técnico

Integrar detección, procesamiento y asistencia en una sola solución tecnológica aplicada a situaciones reales de seguridad.

Resumen técnico

El sistema puede entenderse como una cadena simple: detectar, procesar, alertar y guiar.

Entrada del sistema

  • Lectura del ambiente
  • Detección de gas
  • Detección de monóxido
  • Datos recibidos por sensores

Salida del sistema

  • Alarma local
  • Notificación en la app
  • Indicación para el usuario
  • Respuesta según el tipo de riesgo
